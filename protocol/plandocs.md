# Plandocs

Plandocs are bounded execution documents.

They specify:

- outcome
- owner
- design anchor
- current state
- deliverables
- landing slice
- explicit deferrals
- implementation steps
- verification
- rollback
- evidence
- closeout state

Every non-trivial plandoc must be anchored to a canonical design doc before it
can become approved, active, QA, or resolved.

One plandoc should have one primary outcome. If a document contains independent
outcomes, owners, proofs, or rollback meanings, split it or promote the
coordinating artifact to a milestone with bounded subplans.
