# Protocol Canon

The canonical protocol lives here first. Tool-specific skills adapt this canon
for Codex or Hermes.

Start with `INDEX.md` for the authority boundary and document map.

## Core Documents

- `plandocs.md`: execution document contract.
- `design-docs.md`: architectural blueprint and design-anchor contract.
- `audit.md`: deep audit/refinement behavior.
- `control-boards.md`: control-board artifact shape, not live routing logic.
- `durable-comms.md`: durable handoff artifact shape.
- `github-issues.md`: GitHub ticket intake, proof, comment, and closeout rules.

## Adapter Rule

If a Codex or Hermes skill needs wording that changes protocol meaning, update
the canon here first, then update adapters.
